\chapter{Introduction}
\label{ch:introduction}

The Internet was designed to deliver packets, not experiences. A router forwards a datagram
toward its destination without any notion of \emph{why} the datagram was sent: whether it
carries a frame of an interactive video call that will be worthless if it arrives late, a
byte of a background software update that only cares about aggregate throughput, or a
control message whose loss would be catastrophic[cite: 1]. For decades this deliberate ignorance --
the end-to-end principle -- was a feature[cite: 1]. Yet as applications have become more diverse and
more demanding, the mismatch between what the network optimizes and what the application
actually values has grown into one of the central inefficiencies of modern networking[cite: 1].

Path-aware Internet architectures such as SCION~\cite{Perrig2017} change the terms of this
problem[cite: 1]. In SCION, endpoints discover \emph{multiple} end-to-end paths to a destination and
choose among them, rather than deferring blindly to whatever route BGP happens to install[cite: 1].
This turns path selection from a network-internal decision into an application-facing one:
for the first time the entity that knows the application's intent -- the endpoint -- also
holds the lever that determines how traffic is carried[cite: 1]. The question this thesis studies is
the natural consequence of that shift:

\begin{quote}
\itshape How should an endpoint choose the best path -- or the best \emph{combination} of
paths -- given both the intention of the application and the current state of the network?[cite: 1]
\end{quote}

Answering this question is hard for three compounding reasons[cite: 1]. First, \emph{``best'' is not
a fixed target}[cite: 1]. A path that is best for a bulk transfer (high goodput, tolerant of latency)
is often a poor choice for a video call (which prizes low latency and jitter and will trade
raw throughput to get them)[cite: 1]. Any single hand-tuned objective is therefore wrong for some
application, and training a separate policy for every conceivable objective does not scale[cite: 1].
Second, \emph{the decision space is dynamic and unbounded}[cite: 1]. The set of available paths between
two hosts differs from pair to pair and changes over time as links appear, fail, congest, or
recover; a policy that assumes a fixed, enumerated action set cannot cope[cite: 1]. Third, \emph{the
state is only partially and expensively observable}[cite: 1]. Fresh per-path measurements cost probing
overhead, so a good policy must decide well while measuring little[cite: 1].

This thesis addresses these challenges through a unified principle inspired by recent work in
congestion control[cite: 1]. The Mortise system~\cite{Shen2026} shows that network mechanisms can be
steered to maximize an application's quality of experience (QoE) by treating the application's
\emph{intent} as an explicit input that reshapes how the mechanism is tuned, rather than as a fixed
constant baked into the algorithm[cite: 1]. We adopt this stance and push it across two sequential
levels of operational control[cite: 1].

\paragraph{Motivating Foundation: Intent-Aware Single-Path Selection.} We first investigate
how application intent can modulate discrete path choice for general network flows[cite: 1]. Building
on the DQN-based SCION path selector of Bourak~et~al.~\cite{Bourak2025}, we introduce a
permutation-equivariant \emph{path-scoring} architecture that supports dynamic, dynamic path sets[cite: 1].
Crucially, we condition the agent on application intent using \emph{feature-wise linear modulation}
(FiLM)~\cite{Perez2018}[cite: 1]. We demonstrate analytically and empirically that naive feature concatenation
fails because additive signals cancel under \emph{argmax} selection, whereas FiLM modulation successfully
re-ranks path features to serve diverse application profiles without retraining[cite: 1]. However, this study
also exposes a fundamental boundary condition: when network congestion surges or link quality fluctuates,
even the optimal single path reaches an architectural performance ceiling[cite: 1].

\paragraph{Joint Application--Network Optimization over Multipath.} Driven by the limitations of
single-path selection, we expand our framework to co-optimize both the application and the network
simultaneously[cite: 1]. Leveraging SCION's native path multiplicity and Multipath-QUIC~\cite{DeConinck2017}
transport, we address interactive real-time video streaming by controlling both \emph{what} the
application produces (the video encoder's target bitrate) and \emph{how} the network carries it
(the continuous per-path traffic split)[cite: 1]. We design a two-timescale hierarchical reinforcement-learning
system comprising a slow application agent for bitrate adaptation and a fast transport agent for path
scheduling[cite: 1]. This closes the loop from high-level intent down to application generation[cite: 1].
Our central finding is that the value of learned multipath scheduling depends sharply on network
volatility -- nearly negligible when paths are stable, but decisive under path churn and congestion
bursts, where adapting the application alone collapses[cite: 1].

Throughout, a set of questions guides the reader: \emph{Can one policy serve many intents
without retraining, and why does modulation succeed where concatenation fails?}[cite: 1] \emph{What are
the exact boundaries where single-path selection falls short?} \emph{When is it worth splitting
traffic across paths, and does co-optimizing the application with the network beat optimizing either alone?}[cite: 1]
And \emph{how far are these techniques from practical deployment?}[cite: 1]

\section{Contributions}
\label{sec:intro:contributions}

This thesis makes the following contributions[cite: 1]:

\begin{enumerate}
  \item \textbf{An intent-conditioned single-path selection framework for SCION.} We extend the DQN path
  selector of Bourak~et~al.~\cite{Bourak2025} with (i) a permutation-equivariant path-scoring network
  supporting variable candidate path sets, and (ii) a FiLM-based conditioning mechanism that enables a
  single policy to realize a spectrum of application intents without retraining (\cref{ch:single_path})[cite: 1].

  \item \textbf{Theoretical and empirical evidence for modulative intent conditioning.} We show why
  naively concatenating an intent vector to the state fails in comparison-based selection, proving that
  feature-wise modulation is required to re-order candidate actions (\cref{ch:single_path})[cite: 1].

  \item \textbf{Identification of the single-path performance ceiling.} We characterize the physical
  and operational limits of intent-driven single-path selection under heavy link utilization, establishing
  the empirical bridge to multipath co-optimization (\cref{ch:single_path})[cite: 1].

  \item \textbf{A hierarchical agent pair for joint application--network multipath co-optimization.}
  We design and implement a two-timescale reinforcement-learning system that co-optimizes real-time video
  bitrate (application) and per-path traffic splits (network) over Multipath~QUIC across both an NS-3
  packet simulator and an analytical testbed (\cref{ch:multipath})[cite: 1].

  \item \textbf{An evaluation of the regimes of value for learned network control.} We demonstrate that
  learned multipath splitting yields diminishing returns in static environments but becomes decisive under
  network volatility, path churn, and transient congestion bursts (\cref{ch:multipath,ch:analysis})[cite: 1].
\end{enumerate}

\section{Organization}
\label{sec:intro:organization}

The remainder of this thesis is organized as follows[cite: 1]:

\Cref{ch:background} reviews the technical foundations, including the SCION architecture, goal-conditioned
reinforcement learning, FiLM modulation, and real-time video transport over Multipath QUIC[cite: 1].
\Cref{ch:problem} formalizes the decision problems, system models, desired properties, and trust assumptions[cite: 1].

\Cref{ch:single_path} presents our motivating study on intent-conditioned single-path selection, detailing
its scoring architecture, FiLM conditioning, experimental evaluation, and performance limits[cite: 1].
\Cref{ch:multipath} expands into joint application--network optimization over multipath, introducing the
hierarchical two-timescale SAC design, the testbed implementation, and evaluation across volatile network regimes[cite: 1].

\Cref{ch:synthesis} unifies the findings, analyzing overhead, scalability, generalization, and deployment gaps[cite: 1].
\Cref{ch:related} situates our work within the broader research landscape[cite: 1], and \Cref{ch:conclusion}
concludes with directions for future research[cite: 1].
